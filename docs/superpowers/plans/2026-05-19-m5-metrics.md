# M5 Metrics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Granularity note:** Same as the M4 plan — agent-dispatch + worktree-PR granularity, not per-TDD-step. The playground project's discipline is "every code change = one worktree → one subagent dispatch → one PR". Each implementer subagent does its own TDD decomposition.

**Goal:** Ship M5 (public read-only system-status dashboard at `/metrics`) end-to-end — PRD, ADR-15 + 4 doc amendments, design, GitHub issues, infra (4 new compose containers), backend (`metrics` BC quadruplet), frontend (single `/metrics` page with 19 widgets + 15s polling), and milestone close.

**Architecture:** New `metrics` BC as a four-module quadruplet (`metrics-{api,app,domain,infra}`) — stateless (no Postgres schema). Backend queries `prometheus-playground` via PromQL HTTP API (`/api/v1/query`, `/api/v1/query_range`) and `loki-playground` via LogQL HTTP API (`/loki/api/v1/query_range`). 4 new infra containers (Prometheus + Loki + Alloy + cAdvisor) introduce the observability backbone; Alloy is the single collector that scrapes BC actuator endpoints + cAdvisor + host metrics, and ships docker logs to Loki. Frontend `/metrics` route is a public page (per ADR-09) consuming `GET /api/metrics/dashboard?range=Xh` every 15 s. Logs API is auth-only (ADR-09 amendment) with no P0 UI.

**Tech Stack:**
- **Backend:** Spring Boot 3.3.x, Spring WebFlux (`WebClient` for Prometheus/Loki), Spring Modulith (BC scaffold, no events). No Postgres schema; no JPA in `metrics-infra`. Redisson for rate-limit token bucket.
- **Frontend:** Next.js 14 App Router, **Recharts** (working assumption — ADR-15 confirms) for line charts + sparklines, `EventSource` not used (15s polling sufficient), design tokens reused from M1/M2/M4.
- **Infra (NEW):** `prom/prometheus:v2.54.x`, `grafana/loki:3.x`, `grafana/alloy:1.x`, `gcr.io/cadvisor/cadvisor:v0.49.x`. Exact pins land in ADR-15. cAdvisor + Alloy need read-only `/var/run/docker.sock` mount; security note in `docs/infra-requirements/be.md`.
